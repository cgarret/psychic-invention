## Organization Pattern

The vault leans on MOC-style navigation and dataview-driven inventories.
The naming and filtering model in [MOC Books.md](MOC%20Books.md) suggests chapter notes are stored as structured markdown files with properties like level, module, book, and chapter number.

## Notable Conventions

- Chapter inventories are built from metadata rather than manual lists.
- Imported narrative material is grouped by book and chapter.
- Supporting tools and scripts live alongside the vault content.
- The MOC layer is split between a global vault home and narrower topic-specific hubs.
