## Organization Pattern

The vault is organized as a general-purpose knowledge workspace rather than a single content type.
Topic hubs live in [NOTES_BOX/](./NOTES_BOX/), while the Python files support conversion and testing workflows.

## Notable Conventions

- The top-level MOC file is minimal and acts as a placeholder entry point.
- Topic hubs are kept inside `NOTES_BOX` instead of a separate MOC directory.
- The vault includes Python tooling alongside the notes.
- Imported clippings and research material are kept separate from working notes.
